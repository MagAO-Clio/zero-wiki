What's here:

All FITS files with no stars or targets in the image, as found by searching for FITS files with the AO loop open (AO off), then deleting open-loop user images, and also deleting all identifying info from FITS headers.
A table and text file lists the contents of each date's FITS files, which can be downloaded as gzipped-FITS in a zipped-up directory. Contact Katie Morzinski for help.

NOTES:

  • I've ordered the files by timestamp so that you can see if the Flats were taken at the beginning or the end of the night (times are in UT).
  • I've listed the filter and camera so that you can decide if it's the right Flat or Dark.
  • I've listed the x-y-z dimensions, integration time, and coadds so that you can determine if it's the right Dark, or if it will work to create a good Flat.
  • I've listed the average number of counts per coadd (or per frame in cubes) for each file, so that you can assess whether it is a useful "lights-on" or "lights-off" type Flat.
  • I've listed the detector temperature (keyword TEMP1 in header) and the optics temperature (TEMP2) so that you can compare them to the temperature of your data, to decide whether it's a valid Dark for your data.

NOTE: GOOD/BAD/OK designations of Flat data sets is according to the method I learned from Bruce Macintosh & Henry Roe of "Lights-on" minus "Lights-off" flats (at Keck where there are dome lamps) -- so to do it with Twilight Flats you want a set of brighter ones = "lights-on" and a set of darker ones = "lights-off" but with the same integration time so that you can subtract off the dome/sky thermal floor of scattered light from the on-axis flatfield.
Users may have other ways they wish to build flats, and may have their own opinions as to GOOD/BAD/OK.




How to prepare files for this page (note to self:)

Use write_clio_summary_calibrations.pro :
It reads in all the data for a particular directory, and only writes a summary of the files with AO loop open. It also deletes all identifying info from header, including all telescope pointing info, all user info, all catalog info.
Then check that file and cull it for user's data that was when the loop was open.
And post the zipped-FITS files here, remaining. These are the calibration files. (500 MB limit)
